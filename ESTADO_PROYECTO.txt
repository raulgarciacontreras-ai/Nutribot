1. DESCRIPCION GENERAL
------------------------------------------------------------------------
Nutribot es un bot de Telegram con IA para seguimiento nutricional
personalizado de Nathalie. Usa Gemini 1.5 Flash (gratis) como LLM,
ChromaDB local como vector store para RAG, SQLite para memoria, y
APScheduler para recordatorios automaticos.

Identidad:
  - Nombre del bot: Nutribot (constante BOT_NAME en config.py)
  - Nombre del usuario: Nathalie (constante NATHALIE_NAME en config.py)

Capacidades:
  - Conversacion nutricional por texto
  - Analisis de fotos (refrigerador, platos) con Gemini Vision
  - RAG sobre guia nutricional personalizada
  - Recordatorios automaticos en 5 franjas horarias
  - Memoria conversacional persistente (SQLite)
  - Sticker/meme al final de cada respuesta segun tono

"""
main.py -- Punto de entrada de Nutribot.
Arranca: Telegram bot (polling) + APScheduler + ingesta inicial de RAG.
"""
import asyncio
import logging
import os

from telegram.ext import Application
from config import TELEGRAM_BOT_TOKEN
from bot.telegram_handler import register_handlers
from media.sticker_picker import ensure_folders
from rag.vector_store import is_populated
from scheduler.reminder_scheduler import setup as setup_scheduler

logging.basicConfig(
    format="%(asctime)s | %(levelname)s | %(name)s | %(message)s",
    level=logging.INFO,
)
logger = logging.getLogger(__name__)

def bootstrap() -> None:
    os.makedirs("./data", exist_ok=True)
    ensure_folders()
    if not is_populated():
        logger.warning(
            "ChromaDB vacio. Ejecuta primero: python scripts/ingest_guide.py\n"
            "El bot funcionara sin RAG hasta que lo hagas."
        )

async def main() -> None:
    bootstrap()
    app = Application.builder().token(TELEGRAM_BOT_TOKEN).build()
    register_handlers(app)
    scheduler = setup_scheduler(app)
    scheduler.start()
    logger.info("Scheduler arrancado con %d jobs", len(scheduler.get_jobs()))
    await app.initialize()
    await app.start()
    await app.updater.start_polling(allowed_updates=["message"])
    logger.info("Nutribot arrancado. Esperando mensajes de Nathalie...")
    try:
        await asyncio.Event().wait()
    except (KeyboardInterrupt, SystemExit):
        logger.info("Apagando...")
    finally:
        scheduler.shutdown(wait=False)
        await app.updater.stop()
        await app.stop()
        await app.shutdown()

if __name__ == "__main__":
    asyncio.run(main())

"""
Telegram Handler -- recibe mensajes y fotos, orquesta la respuesta.
Es el puente entre Telegram <-> LLM <-> RAG <-> Memory <-> Stickers.
"""
import logging
from telegram import Update
from telegram.ext import (
    Application, CommandHandler, MessageHandler, ContextTypes, filters,
)
from config import NATHALIE_CHAT_ID, BOT_NAME, NATHALIE_NAME
from memory.store import (
    save_turn, get_recent_turns, get_profile, update_profile,
    get_today_meals, log_meal,
)
from rag.vector_store import retrieve
from llm.gemini_client import build_prompt, chat, chat_with_image, classify_tone
from media.sticker_picker import pick

logger = logging.getLogger(__name__)

def _is_nathalie(update: Update) -> bool:
    return update.effective_chat.id == NATHALIE_CHAT_ID

--- Comandos ---
/start  -> Saludo: "Hola {NATHALIE_NAME}! Soy {BOT_NAME}, tu asistente nutricional personal."
/perfil -> Muestra perfil nutricional de la DB
/help   -> Lista de capacidades

--- Handlers ---
handle_text:  texto libre -> RAG retrieve -> build_prompt -> chat -> classify_tone -> pick sticker -> reply
handle_photo: descarga foto -> RAG retrieve -> build_prompt -> chat_with_image -> classify_tone -> pick sticker -> reply

--- Utilidad ---
_send_media: envia sticker_id o photo segun lo que devuelva el picker
register_handlers: registra todos los handlers en la app de Telegram

Flujo completo por mensaje:
  1. Verificar que es Nathalie (_is_nathalie)
  2. Guardar turno del usuario (save_turn)
  3. Buscar contexto RAG relevante (retrieve)
  4. Armar prompt enriquecido (build_prompt con perfil + comidas de hoy + historial + RAG)
  5. Llamar a Gemini (chat o chat_with_image)
  6. Guardar respuesta (save_turn)
  7. Enviar texto + clasificar tono + enviar sticker

8. OTROS ARCHIVOS CLAVE
------------------------------------------------------------------------

--- rag/vector_store.py ---
ChromaDB local con embeddings de Gemini (text-embedding-004, gratis).
Funciones: is_populated(), get_count(), ingest_chunks(), retrieve()
Procesa en batches de 50 para respetar limites de API.

--- memory/store.py ---
SQLite puro (sin ORM). 3 tablas:
  user_profile:  id, name, weight_kg, height_cm, age, activity, goal, restrictions, symptoms, notes
  conversation:  id, role, content, msg_type, created_at
  meal_log:      id, meal_type, description, est_cal, created_at
Funciones: init_db(), get_profile(), update_profile(), save_turn(), get_recent_turns(),
           log_meal(), get_today_meals()

--- llm/gemini_client.py ---
SYSTEM_PROMPT usa BOT_NAME y NATHALIE_NAME de config.
Reglas clave: nunca sugerir restricciones, celebrar cada comida, cariño sin culpa.
Funciones: build_prompt(), chat(), chat_with_image(), classify_tone()
classify_tone usa gemini-1.5-flash para etiquetar el tono en 1 palabra.

--- media/sticker_picker.py ---
6 categorias: motivacional, gracioso, nutricion, felicitacion, empujoncito, default
Prioridad: file_id de Telegram > archivo local > fallback a default > fallback global
Funcion principal: pick(tone) -> {"type": "sticker_id"|"photo"|"none", "value": ...}

--- scripts/ingest_guide.py ---
Lee guia nutricional (txt/md/pdf), divide en chunks, indexa en ChromaDB.
Soporta PDF via PyMuPDF.

--- setup_check.py ---
6 checks automaticos:
  1. Variables de entorno (.env)
  2. Token de Telegram (valida con getMe)
  3. API de Gemini (genera respuesta de prueba)
  4. ChromaDB local (crea y borra coleccion test)
  5. Guia nutricional indexada (verifica count > 0)
  6. Carpetas de stickers (crea si faltan)
